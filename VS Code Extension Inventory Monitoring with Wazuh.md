# VS Code Extension Inventory Monitoring with Wazuh

This configuration allows a Windows Wazuh agent to collect information about Visual Studio Code extensions installed under local Windows user profiles. The collected extension data is sent to the Wazuh manager, where it is decoded, converted into alerts, and can be used to build an extension inventory dashboard.

The Wazuh agent runs a PowerShell command at a configured interval. Each detected extension is converted into a JSON event and sent to the Wazuh manager for decoding and analysis.

## How It Works

1. The Windows Wazuh agent scans all non-special user profiles under `C:\Users`.
2. It checks the following extension directories:
   - Visual Studio Code Stable: `.vscode\extensions`
   - Visual Studio Code Insiders: `.vscode-insiders\extensions`
3. It reads the `package.json` file from each extension directory.
4. It creates one JSON event for each installed extension.
5. The event is sent to the Wazuh manager.
6. A custom decoder extracts the JSON fields.
7. A custom rule generates an alert for every detected VS Code extension.
8. The generated alerts can be used to create a custom alert based inventory dashboard.

# 1. Configure the Windows Wazuh Agent

Edit the Wazuh agent configuration file:

```text
C:\Program Files (x86)\ossec-agent\ossec.conf
```

Add the following block inside the main `<ossec_config>` section:

```xml
<localfile>
  <log_format>command</log_format>

  <command>powershell.exe -NoProfile -NonInteractive -ExecutionPolicy Bypass -Command "$scanTime=(Get-Date).ToUniversalTime().ToString('o'); Get-CimInstance Win32_UserProfile | Where-Object { -not $_.Special -and $_.LocalPath -like 'C:\Users\*' -and (Test-Path $_.LocalPath) } | ForEach-Object { $profile=$_; $username=Split-Path $profile.LocalPath -Leaf; @(@{edition='stable';path=(Join-Path $profile.LocalPath '.vscode\extensions')},@{edition='insiders';path=(Join-Path $profile.LocalPath '.vscode-insiders\extensions')}) | ForEach-Object { $location=$_; if(Test-Path $location.path){ Get-ChildItem -Path $location.path -Directory -ErrorAction SilentlyContinue | ForEach-Object { $extensionDirectory=$_; $manifestPath=Join-Path $extensionDirectory.FullName 'package.json'; if(Test-Path $manifestPath){ try { $manifest=Get-Content -Path $manifestPath -Raw -ErrorAction Stop | ConvertFrom-Json -ErrorAction Stop; if($manifest.publisher -and $manifest.name){ [ordered]@{integration='vscode_extension_inventory';event_type='extension_inventory';scan_time=$scanTime;hostname=$env:COMPUTERNAME;username=$username;vscode_edition=$location.edition;extension_id=($manifest.publisher+'.'+$manifest.name).ToLower();publisher=[string]$manifest.publisher;extension_name=[string]$manifest.name;display_name=[string]$manifest.displayName;version=[string]$manifest.version;extension_path=$extensionDirectory.FullName} | ConvertTo-Json -Compress } } catch {} } } } } }"</command>

  <alias>vscode_extension_inventory</alias>
  <frequency>21600</frequency>
  <out_format>$(log)</out_format>
</localfile>
```

## Frequency

The `<frequency>` value is specified in seconds:

```xml
<frequency>21600</frequency>
```

`21600` seconds equals **6 hours**.

Running this command too frequently is unnecessary because extension inventories normally change infrequently. For most environments, running the scan every 6, 12, or 24 hours is sufficient.

The frequency can be changed based on your monitoring requirements.

# 2. Add the Custom Decoder on the Wazuh Manager

Edit or create the custom decoder file:

```bash
vi /var/ossec/etc/decoders/local_decoder.xml
```

Add the following decoder:

```xml
<decoder name="vscode-extension-inventory">
  <parent>ossec</parent>
  <prematch type="pcre2">^ossec: output: 'vscode_extension_inventory':\s*</prematch>
  <plugin_decoder offset="after_prematch">JSON_Decoder</plugin_decoder>
</decoder>
```

The decoder matches command output generated with the following alias:

```text
vscode_extension_inventory
```

It then passes the remaining JSON content to the Wazuh JSON decoder.

---

# 3. Add the Custom Rule on the Wazuh Manager

Edit or create the custom rules file:

```bash
vi /var/ossec/etc/rules/local_rules.xml
```

Add the following rule:

```xml
<group name="vscode_extension_inventory,software_inventory,">

  <rule id="107580" level="3">
    <decoded_as>ossec</decoded_as>
    <field name="integration">^vscode_extension_inventory$</field>
    <field name="event_type">^extension_inventory$</field>
    <description>VS Code extension $(extension_id), version $(version), found for user $(username).</description>
    <options>no_full_log</options>
  </rule>

</group>
```

This rule verifies that:

- `integration` equals `vscode_extension_inventory`
- `event_type` equals `extension_inventory`

An alert is generated for every VS Code extension reported by the Windows endpoint.

> **Important:** This rule generates an alert for every installed extension during every inventory scan. In environments with many endpoints or many installed extensions, this can create repeated alerts and increase index usage.

For large environments, consider reducing the scan frequency, lowering the rule level, or using a dedicated index or inventory workflow.

---

# 4. Restart the Wazuh Manager

After adding the decoder and rule, restart the Wazuh manager:

```bash
systemctl restart wazuh-manager
```

Check the service status:

```bash
systemctl status wazuh-manager --no-pager
```

# 7. Create a Custom Dashboard

You can create a custom dashboard by following the Wazuh custom dashboard documentation listed in the References section.

You can also import a prebuilt `.ndjson` dashboard file and use the same dashboard in your environment. To create the `.ndjson` file, copy the JSON content provided in the **Custom Dashboard File** section below, save it with an `.ndjson` extension, and then import it into the Wazuh dashboard.

## Import the `.ndjson` Dashboard File

1. Open the Wazuh dashboard.
2. Click the hamburger menu.
3. Navigate to:

   ```text
   Dashboard Management > Saved Objects
   ```

4. Click the **Import** icon in the upper-right corner.
5. Select the provided `.ndjson` file.
6. Click **Import**.
7. Open the **Dashboards** section and select the imported dashboard.

The dashboard can then be used to review installed VS Code extensions by endpoint, user, publisher, version, and extension ID.
